Windows 비공식(self-signed) 설치 가이드

이 가이드는 비공식(self-signed) 빌드 전용입니다. 정식 배포(signed)는 다운로드 후 더블클릭만으로 설치되므로 이 절차가 필요 없습니다. 비공식 빌드는 테스트 인증서로 서명되어 있어, PC가 이 인증서를 신뢰하도록 등록한 뒤에만 설치할 수 있으며 등록에는 관리자 권한이 필요합니다. 인증서 등록 없이 APPX를 먼저 실행하면 0x800B0109 오류가 발생합니다.

방법 1 · 원클릭 설치 (권장)

  1. 시작 버튼을 우클릭하고 터미널(또는 Windows PowerShell)을 엽니다.
  2. 아래 명령을 붙여넣고 Enter를 누릅니다. 다운로드(약 1.6GB)와 SHA-256 검증이 자동으로 진행됩니다.
  3. 파란 관리자 권한(사용자 계정 컨트롤) 창이 뜨면 "예"를 누릅니다.
  4. "설치 완료"가 표시되면 시작 메뉴에서 DeepWork를 실행합니다.
irm https://braincrew-lab.github.io/deepwork-public/install-windows.ps1 -OutFile "$env:TEMP\deepwork-install.ps1"; powershell -NoProfile -ExecutionPolicy Bypass -File "$env:TEMP\deepwork-install.ps1"

스크립트는 위 카드의 APPX와 테스트 인증서를 내려받아 검증한 뒤 설치합니다. 내용은 install-windows.ps1에서 그대로 확인할 수 있습니다.

방법 2 · 수동 설치

  1. 위에서 APPX와 테스트 인증서(.cer)를 모두 다운로드합니다.
  2. .cer 파일을 더블클릭 → 인증서 설치 → 저장 위치 "로컬 컴퓨터" → "모든 인증서를 다음 저장소에 저장" → 찾아보기에서 "신뢰할 수 있는 루트 인증 기관" 선택 → 마침.
  3. 같은 .cer 파일을 한 번 더 더블클릭해, 같은 방법으로 이번에는 "신뢰할 수 있는 사용자" 저장소에 설치합니다. (두 저장소 모두 등록해야 설치가 통과합니다.)
  4. APPX 파일을 더블클릭하고 앱 설치 관리자에서 설치를 누릅니다.

테스트 인증서를 "신뢰할 수 있는 루트 인증 기관"에 등록하면 이 인증서로 서명된 모든 프로그램을 PC가 신뢰하게 됩니다. 비공식 빌드는 사내 테스트 PC에만 설치하는 것을 권장합니다. 일반 사용자는 이 절차가 필요 없는 정식 배포(signed)를 사용하세요.
